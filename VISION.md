# Опис бачення проєкту "База даних для туристичних маршрутів"

## 1. Назва та мета
**Назва:** База даних для туристичних маршрутів та екскурсій.
**Моя мета:** Створити зручну електронну систему, яка замінить Excel-таблиці та паперові записи в туристичному агентстві. Потрібно, щоб уся інформація про тури, клієнтів і бронювання була в одному місці — у базі даних.

## 2. Основна проблема
Раніше всі дані про тури, гідів та клієнтів зберігалися окремо, що призводило до плутанини і помилок. Моя система має це виправити, забезпечивши **надійне зберігання** та **швидкий пошук**.

## 3. Хто буде користуватися системою (Користувачі)
1.  **Адміністратор:** Може створювати нові тури, додавати локації, реєструвати гідів та дивитися загальну статистику.
2.  **Менеджер:** Приймає заявки від клієнтів, підтверджує або скасовує бронювання.
3.  **Клієнт:** Може обрати тур, забронювати місце та залишити відгук.

## 4. Технології, які я використовувала
* **База даних:** MySQL (бо вона надійна та швидка).
* **Веб-частина:** Python (Flask) та SQLAlchemy для зручного підключення до бази.
